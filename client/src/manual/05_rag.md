# Data Base（RAG管理）

RAG（Retrieval-Augmented Generation：検索拡張生成）用のドキュメントデータベースを管理するページです。
ドキュメントをアップロードすることで、AIエージェントがその内容を参照しながら議論できるようになります。

## RAGとは

RAG（検索拡張生成）は、AIが議論する際に関連ドキュメントを自動的に参照する仕組みです。

- アップロードしたドキュメントをベクトルデータベースに保存
- 議論中に関連する内容を自動検索・参照
- ペルソナが持つ知識として機能する

**活用例：**
- 社内資料・仕様書を読み込ませて、その内容を踏まえた議論を行う
- 競合他社の情報を与えて、競合分析の議論をさせる
- 過去の会議録を与えて、継続的な議論を行う

## 画面の構成（3タブ）

### Manage タブ
ドキュメントのアップロードと初期化を行います。

#### コレクションの初期化
- **タグ名** : ドキュメントのグループ名（例：`ProjectAlpha`、`Research2024`）
- タグ名でドキュメントを分類・管理できます
- 既存のタグを選択、または新しいタグ名を入力して初期化します

#### ドキュメントのアップロード
- テキストファイル（`.txt`）やPDF（`.pdf`）などをアップロードできます
- アップロード時にチャンク戦略を選択します：

| チャンク戦略 | 説明 |
|---|---|
| Fixed Size | 固定サイズで分割（高速、シンプル） |
| Semantic | 意味的なまとまりで分割（精度が高い） |
| Paragraph | 段落で分割（文書の構造を保持） |

### Browse タブ
アップロード済みのドキュメントチャンクを閲覧・管理します。

- タグでフィルタリングして表示
- 各チャンクの内容を確認
- 不要なチャンクを個別に削除

### Playground タブ
RAGの検索動作をテストできます。

- 検索クエリを入力してヒットするチャンクを確認
- RAGが正しく機能しているかを事前に確認できます

## ペルソナへのRAG紐付け

Personasページで各ペルソナにRAGコレクション（タグ）を紐付けることができます。

設定方法：
1. Personasページでペルソナを編集
2. **RAG設定** セクションでRAG種別とタグを選択
3. 保存するとペルソナがそのコレクションを参照するようになる

---

## 注意事項

- RAGデータベースにはQdrantを使用しています
- ドキュメントの内容は埋め込みベクトルとして保存されます
- 大量のドキュメントをアップロードするとインデックス作成に時間がかかる場合があります
