# ストラテジー一覧

**テーマ内ストラテジー**は、1つのテーマ内でエージェントがどのように発言を進めるかを決めるパターンです。

← [New Session の設定に戻る](page:setup)

---

## ストラテジー一覧

### シーケンシャル（バトンリレー）
順番に発言し、前の人の出力を次の人に渡していく形式です。

- **特徴**: 前のエージェントの発言を引き継いで積み上げる
- **向いている場面**: 段階的に議論を発展させたい時、リレー形式で考えを深めたい時
- **例**: A が骨子を作成 → B が肉付け → C が仕上げ

---

### 並列独立（ブレスト）
各エージェントが独立して意見を出し、ファシリテーターが最後に集約します。

- **特徴**: お互いの意見に影響されないため、多様なアイデアが生まれやすい
- **向いている場面**: ブレインストーミング、アイデア出し
- **例**: 3人が各自で企画案を提案 → ファシリテーターがまとめる

---

### ラウンドロビン（ディベート）
全員が順番に発言するラウンドを複数回繰り返します。

- **特徴**: 同じループを繰り返すことで議論が深まる
- **向いている場面**: 議論を深掘りしたい時、各ターンで意見が洗練されていくイメージ
- **例**: 1周目：各自の意見 → 2周目：相互へのフィードバック → 3周目：最終意見

---

### 階層型（計画・実行・反省）
マネージャーが計画し、ワーカーが実行し、その結果を評価・修正するサイクルです。

- **特徴**: 役割分担が明確で、品質管理サイクルが自然に生まれる
- **向いている場面**: 品質を高めたい時、計画と実行のフェーズが重要な時
- **例**: マネージャーが方針策定 → ワーカーA・Bが実行 → レビュアーが評価・改善指示

---

### 敵対的（生成・批判）
生成役が提案し、批判役がダメ出しし、修正を繰り返すパターンです。

- **特徴**: 弱点を徹底的に洗い出すことで、提案の堅牢性が高まる
- **向いている場面**: 提案の穴を潰したい時、反論を想定したい時
- **例**: 提案者が企画案提出 → 批判者が問題点を指摘 → 提案者が改善案を提出（繰り返し）

---

### 陪審員・裁判官
ディベーターが議論し、裁判官が最終判定を下すパターンです。

- **特徴**: 中立的な評価者が最終決定を行うため、合意形成がしやすい
- **向いている場面**: 複数案から最良を選びたい時、客観的な判断が必要な時
- **例**: 賛成派・反対派がそれぞれ主張 → 裁判官が総合評価して最終案を決定

---

### 動的ルーティング（司会者主導）
司会者が会話の文脈を読み、次に発言させるエージェントを動的に指名します。

- **特徴**: 状況に応じて柔軟に進行できる、最も自由度が高いパターン
- **向いている場面**: 議論の流れを読みながら柔軟に進行させたい時
- **例**: 司会者が「今の議題はB専門分野なのでBに回す」と判断しながら進行

---

### 分割統治（Map-Reduce）
プランナーが問題を分割し、ワーカーが個別に処理し、サマライザーが統合します。

- **特徴**: 大きなタスクを並列処理できる、効率的なパターン
- **向いている場面**: 大きなタスクを分けて処理したい時、並列作業が可能な時
- **例**: プランナーが3つのサブ課題に分割 → 各ワーカーが担当 → サマライザーが統合

---

### 動的エージェント生成
メタエージェントが議題に最適なペルソナをその場で動的に生成して議論させます。

- **特徴**: 事前にペルソナを用意しなくても、AIが最適な専門家を作り出す
- **向いている場面**: 必要な専門家が事前に分からない時、多様な視点が必要な時
- **例**: メタエージェントが「この議題には法律専門家と財務専門家が必要」と判断して生成

---

## 選び方のまとめ

| やりたいこと | おすすめのストラテジー |
|---|---|
| とりあえず議論させたい | シーケンシャル |
| アイデアをたくさん出したい | 並列独立（ブレスト） |
| 議論を深く掘り下げたい | ラウンドロビン |
| 提案の弱点を潰したい | 敵対的 |
| 品質を段階的に高めたい | 階層型 |
| 複数案から1つを選びたい | 陪審員・裁判官 |
| 柔軟に進行させたい | 動的ルーティング |
| 大きなタスクを分割したい | 分割統治 |
| 必要な専門家が分からない | 動的エージェント生成 |
