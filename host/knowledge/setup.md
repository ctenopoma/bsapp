# セッション設定 入力ガイド

このファイルを編集して、セッション設定ヘルパーの知識を追加してください。

## 基本的な考え方

セッション設定では、AI議論の「何について」「どのように」議論するかを決めます。

- **共通テーマ**: 全テーマに共通する大きな方向性。例：「2030年の社会課題」「新製品の企画」
- **事前情報**: 全エージェントに共有する背景知識。ドキュメントの内容、前提条件、制約事項など
- **テーマ**: 個別の議論テーマ。1セッションに複数設定でき、順番に議論される

## テーマ内ストラテジー（各テーマの議論パターン）

| ストラテジー | 説明 | 向いている場面 |
|---|---|---|
| シーケンシャル（バトンリレー） | 順番に発言し、前の人の結果を次に渡す | 段階的に議論を積み上げたい時 |
| 並列独立（ブレスト） | 各自が独立に意見を出し、ファシリテーターが集約 | アイデア出し、ブレインストーミング |
| ラウンドロビン（ディベート） | 全員が順番に発言するループを複数回繰り返す | 議論を深掘りしたい時 |
| 階層型（計画・実行・反省） | マネージャーが計画→ワーカーが実行→評価・修正を繰り返す | 品質を高めたい時、計画が重要な時 |
| 敵対的（生成・批判） | 生成役が提案し、批判役がダメ出し、修正を繰り返す | 提案の堅牢性を高めたい時 |
| 陪審員・裁判官 | ディベーターが議論し、裁判官が最終判定 | 複数案から最良を選びたい時 |
| 動的ルーティング（司会者主導） | 司会者が文脈を読んで次の発言者を動的に指名 | 柔軟な議論進行が必要な時 |
| 分割統治（Map-Reduce） | プランナーが分割→ワーカーが個別処理→サマライザーが統合 | 大きなタスクを分けて処理したい時 |
| 動的エージェント生成 | メタエージェントが議題に最適なペルソナをその場で生成 | 最適な専門家が分からない時 |

## プロジェクトフロー（テーマ間の進行制御）

| フロー | 説明 | 向いている場面 |
|---|---|---|
| ウォーターフォール | テーマを順番に1つずつ実行 | 基本的な順次実行 |
| ステージゲート | 各テーマ後にゲートキーパーが品質チェック、不合格なら差し戻し | 品質基準がある時 |
| アジャイル/スプリント | 全テーマを複数回繰り返し、完成度を評価 | 反復的に改善したい時 |
| 条件分岐/ツリー | テーマの結論に応じて次のテーマを動的に選択 | 結論によって方向性が変わる時 |
| V字型 | 全テーマを実行後、逆順でレビュー | 前半の要件と後半の実装の整合性確認 |
| ゲーム理論/対立型 | 提案陣営vs批判陣営で議論し、合意形成者が最終案を導く | 対立する視点が必要な時 |
| ブラックボード型 | コーディネーターが共有状態を読み、次の担当を動的に指名 | 複雑な協調作業が必要な時 |
| トーナメント/進化型 | 同じテーマを並列実行し、審査員が最良を選択 | 複数アプローチを比較したい時 |
