# BSapp バックエンド (host) 🚀

BSapp のバックエンドは、**FastAPI** をベースにして構築された、AI多重ペルソナ・ディスカッションアプリのコアサーバーです。外部のクラウドサービス（OpenAIAPIなど）に依存せず、自分自身で構築したローカルの大規模言語モデル（LLM）とベクトルデータベースに完全にアクセスして動作するように設計されています。

---

## ✨ 機能と特徴

- **多重ペルソナ・ディスカッション・システム**: 異なる役割や性格を持たせた複数のAIエージェントの会話と議論を、サーバーサイドで自動進行・オーケストレーションします。
- **ローカル RAG (検索拡張生成)**: ローカルの埋め込みモデル（Embedding）とベクトルデータベース（Qdrant）を使用し、過去の議論の文脈を考慮した高度な発言を生成します。
- **ローカルLLM ネイティブ対応**: OpenAI互換のAPIエンドポイントを提供するローカルLLMサーバー（Ollama、vLLM、LM Studioなど）に完全対応しています。
- **モデルの独立した詳細設定**: 「テキスト生成用 (LLM)」、「埋め込み用 (Embedding)」、「リランク用 (Rerank)」のそれぞれに対して、固有のIPアドレス、ポート番号、モデル名、APIキーを設定でき、非常に柔軟なカスタマイズが可能です。

---

## 📋 必要な前提環境

起動する前に、以下のソフトウェアがインストールされていることを確認してください。

1. **Python 3.13 以上**
2. **[uv](https://docs.astral.sh/uv/)** (非常に高速でモダンなPythonパッケージ管理ツール)
3. **[Qdrant](https://qdrant.tech/)** (ローカルで動かすベクトルデータベース。Docker環境を推奨)
4. [Ollama](https://ollama.com/) などのローカルLLMサーバー

---

## 🛠️ セットアップ手順

### 1. 依存パッケージのインストール

`uv` を使えば、仮想環境の作成から依存ライブラリのインストールまで一瞬で完了します。

```bash
# hostディレクトリ内で実行
uv sync
```

### 2. Qdrant (ベクトルデータベース) の起動

Qdrant はDockerを使って簡単に起動できます。

```bash
docker run -d -p 6333:6333 -p 6334:6334 \
    -v $(pwd)/qdrant_storage:/qdrant/storage:z \
    qdrant/qdrant
```

### 3. ローカルLLMサーバーの起動

お使いのLLMサーバーを立ち上げます。例えば **Ollama** を使う場合、以下のようにサーバーを立ち上げ、必要なモデルを準備してください。

```bash
# Ollamaサーバーを起動 (デフォルトポート: 11434)
ollama serve

# 別のターミナルから必要なモデルをダウンロード (例)
ollama pull llama3             # テキスト生成(会話)用
ollama pull nomic-embed-text   # 埋め込み(ベクトル化)用
```

## ⚙️ 環境変数 (Configuration)

`.env` ファイルを作成（またはコピー）し、各モデルの接続先情報を設定します。

```env
# ==========================================
# ベクトルデータベース (Qdrant) 設定
# ==========================================
QDRANT_URL=http://localhost:6333

# ==========================================
# LLM (テキスト生成: エージェントの発言用) 設定
# ==========================================
LLM_IP=127.0.0.1
LLM_PORT=11434
LLM_MODEL=llama3
LLM_API_KEY=dummy   # Ollamaは"dummy"でOK。プロキシやvLLMを使う場合は実際のキーを指定。

# ==========================================
# Embedding (ドキュメントや文脈のベクトル化用) 設定
# ==========================================
EMBEDDING_IP=127.0.0.1
EMBEDDING_PORT=11434
EMBEDDING_MODEL=nomic-embed-text
EMBEDDING_API_KEY=dummy
VECTOR_SIZE=768     # 使用するモデルの出力次元数に必ず合わせる (nomic-embed-text は 768)

# ==========================================
# Rerank (検索結果の並び替え用: 将来的な拡張機能)
# ==========================================
RERANK_IP=127.0.0.1
RERANK_PORT=11434
RERANK_MODEL=reranker
RERANK_API_KEY=dummy
```

### 💡 設定のポイント：
- **`*_IP` / `*_PORT`**: バックエンド側は、標準的な `/v1` エンドポイント (`http://127.0.0.1:11434/v1`) として各APIへリクエストします。OllamaのAPIはネイティブでこれに互換性があるため、問題なく動作します。
- **`*_API_KEY`**: 認証が必要なプロキシやサーバーを通す場合はここにキーを設定します。

## 🚀 サーバーの起動 (開発用)

FastAPIサーバーをリロードモードで起動するには、以下のコマンドを実行します。

```bash
uv run uvicorn src.main:app --reload --host 0.0.0.0 --port 8000
```

起動後、以下のアドレスからアクセスできます。
- **API ベースURL**: `http://localhost:8000`
- **インタラクティブ API ドキュメント (Swagger UI)**: `http://localhost:8000/docs`

---

## 🏗️ フォルダ・ファイル構成

- `src/main.py` - FastAPIアプリケーションのエンドポイント定義・CORS設定。
- `src/api/` - セッション操作、RAG（追加データ学習）操作などのルーティング定義。
- `src/agent_runner.py` - LangChainを利用したプロンプト作成と推論エンジン、エージェント会話ループ。
- `src/rag_manager.py` - データベースへの自動チャンク分割、ベクトル登録、Qdrant側との通信同期インターフェース。
- `src/session_manager.py` - ペルソナ情報や現在のディスカッションの状況を一時保存するステート管理モジュール。
